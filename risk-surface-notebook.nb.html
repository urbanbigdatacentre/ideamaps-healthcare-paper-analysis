Estimating Risk Surface

Preliminary tasks

Preparation

This notebook estimates a spatial risk surface for maternal deaths in Kano, Nigeria, based on the kernel density ratio method: https://arxiv.org/pdf/1707.06888

Key optimizations applied: - WorldPop raster is aggregated before point generation to reduce memory load - Population points are capped at a maximum sample size (10,000) - Fixed bandwidth used first for speed; adaptive bandwidth optional - Duplicate chunk names fixed - Cases/Controls variable naming corrected

Main Analysis: Maternal Death Risk Surface in Kano

Step 1: Reproject all data to UTM Zone 32N

spatstat’s as.owin() requires a projected coordinate system (metres),

not geographic coordinates (degrees). WGS84 will cause as.owin() to fail.

UTM Zone 32N (EPSG:32632) is appropriate for Kano, Nigeria.

All datasets must use the same CRS for ppp objects to align correctly.

CRS_UTM <- 32632

kano_proj           <- sf::st_transform(kano, crs = CRS_UTM)
maternal_deaths_proj <- sf::st_transform(maternal_deaths, crs = CRS_UTM)

cat("All data reprojected to UTM Zone 32N\n")
All data reprojected to UTM Zone 32N

Step 2: Filter maternal deaths to Kano study area

maternal_deaths contains records from the wider Kano region.

Deaths outside the study area boundary have no corresponding

population Controls and would bias the risk surface.

maternal_deaths_kano <- sf::st_filter(maternal_deaths_proj, kano_proj)

Step 3: Generate population points from WorldPop raster

The original raster_to_random_points() generates points equal to the

population value per cell (mean ~481, max ~8741), producing ~1.8M points

and crashing R.

Fix: scale down all values by SCALE_FACTOR before generating points.

A cell with 500 women gets 5 points, a cell with 8741 gets 87 points.

Relative proportions are preserved so the kernel density estimate is

mathematically equivalent to the original approach.

pmax(1, …) ensures every cell contributes at least 1 point.

Step 4: Build spatial window from Kano polygon boundary

Using the actual Kano boundary polygon (not a bounding box rectangle)

ensures the risk surface matches the study area shape exactly.

This requires projected coordinates - hence the UTM reprojection above.

kano_poly   <- sf::st_cast(kano_proj, "POLYGON")
kano_window <- spatstat.geom::as.owin(kano_poly)

Optimal bandwidth: 4759.528 
rr_geom <- risk(
  f        = deaths_ppp,
  g        = pop_ppp,
  h0       = h0,
  tolerate = TRUE,
  doplot   = TRUE
)

plot(rr_geom, main = "Risk Surface - Geometric Bandwidth")

Option C: Adaptive bandwidth (most accurate, slowest)

Set eval=TRUE in chunk header when ready to run - may take 10-30 mins

hpilot.f <- BOOT.density(deaths_ppp)
hpilot.g <- BOOT.density(pop_ppp)

rr_adapt <- risk(
  f               = deaths_ppp,
  g               = pop_ppp,
  h0              = h0,
  adapt           = TRUE,
  tolerate        = TRUE,
  hp              = c(hpilot.f, hpilot.g),
  pilot.symmetry  = "pooled",
  davies.baddeley = 0.05
)
plot(rr_adapt, main = "Risk Surface - Adaptive Bandwidth")
